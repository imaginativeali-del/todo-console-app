# Phase 5: Cloud Deployment

## Overview
This phase explains how the Todo application can be deployed to the cloud.

## Cloud Strategy
- Application container is pushed to a container registry
- Kubernetes cluster is hosted on a cloud provider
- Load balancer exposes the application publicly

## Possible Cloud Providers
- AWS
- Google Cloud Platform
- Microsoft Azure

## Deployment Steps (High Level)
1. Build application container image
2. Push image to container registry
3. Create Kubernetes cluster
4. Apply deployment configuration
5. Expose service using load balancer

## Status
Cloud deployment strategy is documented.
Live deployment can be performed when infrastructure is available.
